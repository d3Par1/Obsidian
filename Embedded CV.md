## Етап 4 — Технічна співбесіда (тут зазвичай "ріжуть")

### Теми для embedded junior — мінімум, що треба знати:

#### A. C / C++ fundamentals (90% питань)
- [ ] **Pointers vs references** — різниця, коли що використовувати
- [ ] **Memory: stack vs heap** — як організована пам'ять процесу
- [ ] **`malloc/free` vs `new/delete`** — і чому в embedded часто **уникають** динамічної пам'яті
- [ ] **`const`, `volatile`, `static`** — особливо `volatile` для регістрів!
- [ ] **Struct padding/alignment** — чому `sizeof(struct)` не дорівнює сумі полів
- [ ] **Bit manipulation** — `&`, `|`, `^`, `<<`, `>>`, маски, `BIT(n)`
- [ ] **Endianness** — little vs big, як перевірити, як конвертувати
- [ ] **Function pointers** — синтаксис, callback patterns

#### B. Embedded specifics
- [ ] **Polling vs interrupts** — коли що використовувати
- [ ] **ISR (Interrupt Service Routine)** — правила: short, no malloc, no printf, `volatile` змінні
- [ ] **UART / I²C / SPI** — базова відмінність, baud rate, master/slave
- [ ] **GPIO modes** — input/output, pull-up/pull-down, open-drain
- [ ] **Timers** — counter, compare, PWM
- [ ] **DMA** — навіщо потрібен (off-load CPU)
- [ ] **Watchdog timer** — що робить, навіщо

#### C. Linux / OS basics
- [ ] **Process vs thread** — pid, fork(), синхронізація
- [ ] **Signals** — SIGINT, SIGKILL, SIGTERM
- [ ] **File descriptors** — stdin/stdout/stderr, redirection
- [ ] **Permissions** — `chmod`, `chown`, octal notation
- [ ] **systemd unit basics** — як зробити сервіс
- [ ] **Bash:** loops, conditions, `grep`, `awk`, `sed` basics
- [ ] **Networking:** `netstat`, `ss`, `tcpdump` базово

#### D. Алгоритми (lite — для junior)
- [ ] **Big-O** — час і пам'ять, для O(1)/O(log n)/O(n)/O(n²)
- [ ] **Reverse linked list, reverse string** — класика
- [ ] **Two pointers, sliding window** — патерни
- [ ] **Binary search**
- [ ] **Sort: bubble vs quicksort vs mergesort** — concept

#### E. Drone-specific (якщо вакансія дрони)
- [ ] **MAVLink** — message-based protocol, що це і навіщо
- [ ] **PX4 vs ArduPilot** — різниця між двома стеками
- [ ] **PID controller** — концепція (P, I, D — що робить кожна частина)
- [ ] **IMU / GPS / barometer** — які дані дають, чому потрібен sensor fusion
- [ ] **Quaternions vs Euler angles** — на рівні "знаю що таке"

### Live-coding — типові задачі:
1. **"Напиши функцію, що визначає, чи є степінь двійки"** → `n & (n-1) == 0`
2. **"Поверни рядок навпаки in-place"** — two pointers
3. **"Зчитай у циклі дані з UART і запиши в circular buffer"** — embedded класика
4. **"Знайди bug у цьому фрагменті"** — часто `==` замість `=`, missing `volatile`, race condition

### Поведінка на технічці:
- ✅ **Думай вголос** — інтерв'юер хоче бачити процес, не тільки результат
- ✅ **Кажи "не знаю"** замість галюцинацій
- ✅ Перепитуй умову — це показує інженерне мислення
- ❌ Не сиди мовчки 5 хвилин — кажи що думаєш
- ❌ Не оборонистся, якщо знайшли баг — "так, ви праві, виправляю"
