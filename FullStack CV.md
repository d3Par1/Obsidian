## Технічна підготовка — must-know для full-stack junior

### A. JavaScript fundamentals (топ-питання)
- [ ] **`var` / `let` / `const`** — scope, hoisting, temporal dead zone
- [ ] **Closures** — як працюють, типовий приклад `for` + `setTimeout`
- [ ] **`==` vs `===`** — type coercion gotchas
- [ ] **`this` keyword** — у різних контекстах (function, arrow, class, callback)
- [ ] **Promise vs async/await** — error handling, Promise.all/race/allSettled
- [ ] **Event loop** — call stack, microtask queue, macrotask queue (must-know)
- [ ] **Array methods** — `map`, `filter`, `reduce`, `forEach`, `flat`, `flatMap`
- [ ] **Destructuring, spread/rest, optional chaining (`?.`)**
- [ ] **Modules** — CommonJS vs ES Modules, `require` vs `import`

### B. Node.js / Express
- [ ] **Request lifecycle** — middleware chain, error handling, `next()`
- [ ] **Routing** — REST conventions, params vs query vs body
- [ ] **Async patterns** — callback hell → Promises → async/await
- [ ] **Streams** — readable/writable/transform, when to use
- [ ] **Process** — `process.env`, signals, exit codes
- [ ] **Authentication** — sessions vs JWT, cookies, CSRF basics

### C. SQL & databases
- [ ] **SELECT з JOIN** — INNER, LEFT, RIGHT — must understand
- [ ] **GROUP BY + aggregates** — COUNT, SUM, AVG, MAX
- [ ] **Indexes** — коли допомагають, коли шкодять
- [ ] **N+1 проблема** — типовий ORM анти-патерн
- [ ] **Transactions, ACID** — на рівні концепції
- [ ] **NoSQL vs SQL** — коли що краще

### D. HTTP / REST
- [ ] **HTTP methods** — GET / POST / PUT / PATCH / DELETE — semantics
- [ ] **Status codes** — 2xx / 3xx / 4xx / 5xx — основні
- [ ] **Headers** — Content-Type, Authorization, CORS
- [ ] **REST vs RPC vs GraphQL** — концептуально
- [ ] **CORS** — звідки виникає, як вирішується

### E. Frontend basics
- [ ] **HTML semantic tags** — чому важливо для accessibility/SEO
- [ ] **CSS box model, flexbox, grid** — must
- [ ] **DOM events** — bubbling, capturing, delegation
- [ ] **Browser rendering** — critical rendering path basics
- [ ] **localStorage / sessionStorage / cookies** — різниця

### F. Алгоритми (для веб — теж питають, але простіше за embedded)
- [ ] **Big-O** — для всього основного
- [ ] **Hash maps** — як влаштовані, для чого
- [ ] **Sorting** — quicksort/mergesort на рівні концепції
- [ ] **Two pointers, sliding window**
- [ ] **Recursion + memoization**
- [ ] LeetCode — 30–50 Easy задач у JavaScript

## Live-coding на веб співбесіді — типові задачі

1. **"Implement debounce / throttle"** — класика
2. **"Flatten a deeply nested array"** — recursion + array methods
3. **"Reverse a string"** / **"Check palindrome"** — warm-up
4. **"Implement Promise.all from scratch"** — middle-level
5. **"Write a fetch wrapper with retries"** — практична задача
6. **"Build a todo list component"** — UI рівень

## Тестове завдання — типові формати

1. **CRUD-app** — "побудуй простий блог: список постів, створення, редагування, видалення"
2. **API integration** — "візьми public API X, побудуй UI який показує дані з фільтрацією"
3. **Bug-fix** — "у цьому коді є 5 багів, знайди і виправ"
4. **System design lite** — "як би ти спроектував YouTube-like feed?" (рідко для junior)

**Як здавати веб-тестове:**
1. README з інструкцією `npm install && npm start`
2. **Деплой live** — Vercel / Netlify / Render (free tiers) → посилання на робочий додаток
3. Чисті комміти (`feat:`, `fix:`, `refactor:`)
4. Базові тести (Jest для логіки, Playwright для UI)
5. Responsive layout (мобілка теж має працювати)
