#### 📝 Тип 2: "Learning notes" — глибока теорія
```
I finally understood what "volatile" means in embedded C 🎯

Compiler optimizations are great — until they aren't. When a register 
or ISR-modified variable doesn't have `volatile`, the compiler may 
optimize away reads/writes thinking "nothing changes here." Result: 
your code looks correct but your hardware misbehaves.

Rule of thumb:
🔸 Memory-mapped registers → always volatile
🔸 Variables modified in ISR → volatile  
🔸 Multi-threaded shared state → volatile + synchronization

Took me an evening of confused debugging to internalize this.

#embedded #cprogramming #firmware
```

1. **Skills > Headline > About > Posts** — у такому порядку важливості. Skills встановлюються один раз і дають passive discovery весь час.

2. **GitHub link у Featured** — обов'язково. Embedded-recruiter перевірить твій код **до** того, як читатиме CV.

3. **Не перевантажуй "C++ developer"** якщо пишеш embedded — використовуй **"Embedded C++ Engineer"** як точніший keyword.

4. **Постинг про defense-tech motivation** — це твій **diferentiator**. 90% junior'ів — generic, ти будеш помітний.
